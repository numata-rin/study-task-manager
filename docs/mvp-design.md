# Study Task Manager MVP Design

## 1. アプリ概要

Study Task Manager は、React 学習・Django 学習・研究・資格勉強などの学習タスクを管理するためのシンプルなタスク管理アプリである。

MVP段階では、ユーザーが学習タスクを追加・確認・完了切り替え・削除でき、ブラウザをリロードしてもタスクが保持される状態を目指す。

---

## 2. 作成目的

このアプリの主な目的は、React を用いた基本的なフロントエンド開発の演習を積むことにある。

具体的には、以下の項目を自分自身で行えるようにする。

- React コンポーネントを分割して実装できること
- useState を使って状態管理ができること
- フォーム入力を扱えること
- 配列データを一覧表示できること
- ユーザー操作に応じて表示を更新できること
- LocalStorage を使って簡易的なデータ保存ができること
- TypeScript を使ってデータ構造を型で管理できること

---

## 3. MVPのゴール

MVPでは、以下の状態を完成とする。

> タスクを追加・表示・完了切り替え・削除でき、LocalStorageに保存される学習タスク管理アプリ

MVP完成時点で、ユーザーは以下の操作ができる。

- タスクを一覧で確認できる
- 新しいタスクを追加できる
- タスクの完了・未完了を切り替えられる
- 不要なタスクを削除できる
- ページをリロードしてもタスクが保持される

---

## 4. MVPで実装する機能

### 4.1 タスク一覧表示

登録されているタスクを一覧で表示する。

表示する情報は以下とする。

- タスク名
- 完了状態

MVP段階では、カテゴリ・優先度・期限などは表示しない。

---

### 4.2 タスク追加

ユーザーがフォームにタスク名を入力し、新しいタスクを追加できるようにする。

仕様：

- 入力欄にタスク名を入力する
- 追加ボタンを押すとタスク一覧に追加される
- 空文字では追加できない
- 追加後、入力欄は空になる

---

### 4.3 完了状態の切り替え

タスクごとに、完了・未完了を切り替えられるようにする。

仕様：

- 各タスクにチェックボックスを表示する
- チェックすると完了状態になる
- もう一度チェックを外すと未完了に戻る
- 完了済みタスクは見た目で区別できるようにする

例：

- 取り消し線を付ける
- 文字色を薄くする

---

### 4.4 タスク削除

不要になったタスクを削除できるようにする。

仕様：

- 各タスクに削除ボタンを表示する
- 削除ボタンを押すと対象タスクが一覧から消える
- 他のタスクには影響しない
- タスクが0件になってもエラーにならない

---

### 4.5 LocalStorage保存

タスクデータをLocalStorageに保存し、リロード後も保持されるようにする。

仕様：

- 初回表示時にLocalStorageからタスクを読み込む
- 保存データがない場合は初期データを使用する
- タスクが追加・削除・更新されたらLocalStorageに保存する
- リロード後もタスク一覧が維持される

---

## 5. MVPでは実装しない機能

MVPでは、スピード感を優先するため、以下は実装しない。

- ログイン機能
- バックエンド連携
- データベース保存
- タスク編集機能
- カテゴリ設定
- 優先度設定
- 期限設定
- 検索機能
- フィルター機能
- 並び替え機能
- グラフ表示
- 学習時間記録
- 通知機能

これらは、MVP完成後の改善フェーズで検討する。

## 8.コンポーネント設計

MVP段階では、以下のコンポーネント構成を想定する。

```text
src/
├── components/
│   ├── TaskForm.tsx
│   ├── TaskList.tsx
│   └── TaskItem.tsx
├── data/
│   └── initialTasks.ts
├── types/
│   └── task.ts
├── utils/
│   └── localStorage.ts
├── App.tsx
└── main.tsx
```

### App

アプリ全体の状態管理を担当する。

主な役割：

- tasks stateを持つ
- タスク追加処理を持つ
- 完了切り替え処理を持つ
- タスク削除処理を持つ
- LocalStorageとの連携を行う
- 子コンポーネントにデータと関数を渡す

### TaskForm

タスク追加フォームを担当する。

主な役割：

- 入力値を管理する
- フォーム送信時に新しいタスクを追加する
- 空文字入力を防ぐ
- 追加後に入力欄を空にする

### TaskList

タスク一覧全体の表示を担当する。

主な役割：

- tasks配列を受け取る
- タスクがある場合は一覧表示する
- タスクが0件の場合はメッセージを表示する
- 各タスクをTaskItemに渡す

### TaskItem

タスク1件分の表示を担当する。

主な役割：

タスク名を表示する
完了状態を表示する
チェックボックスで完了状態を切り替える
削除ボタンを表示する

## 10.状態管理方針

MVP段階では、ReactのuseStateとuseEffectを使う。

### 使用する主なstate

```text
const [tasks, setTasks] = useState<Task[]>([]);
```

### 方針
- タスク一覧はAppコンポーネントで関ｒに
- 追加・削除・完了切り替えの処理もAppに置く
- 子コンポーネントにはpropsでデータと関数を渡す
- グローバル状態管理ライブラリは使用しない

### 使用しないもの
- Redux
- Zustand
- Context API
- React Query

### 理由：
MVP段階では状態が小さいため、外部ライブラリを使う必要がない。
まずはReact標準の状態管理に慣れることを優先する。

## 11.LocalStorage設計

LocalStorageには、タスク配列をJSON形式で保存する。

### 保存キー

```text
const STORAGE_KEY = "study-task-manager-tasks";
```

### 保存するデータ
```text
Task[]
```

### 保存処理
- tasksが変更されるたびにLocalStorageへ保存する
- 保存時は JSON.stringify を使う

### 読み込み処理
- 初回表示時にLocalStorageから読み込む
- 読み込み時は JSON.parse を使う
- 保存データがない場合は初期データを使用する

## 13.MVP完成条件

以下をすべて満たしたら、MVP完成とする。

- タスク一覧が表示される
- タスクを追加できる
- 空文字ではタスクを追加できない
- タスクの完了状態を切り替えられる
- 完了済みタスクが見た目で分かる
- タスクを削除できる
- タスクが0件でもエラーにならない
- リロード後もタスクが保持される
- TypeScriptエラーが発生していない
- 不要なconsole.logが残っていない

## 15.開発方針

このMVPでは、完璧な設計よりも、まず動くアプリを完成させることを優先する。

開発では以下を意識する。

- 小さなIssue単位で作業する
- 1つのIssueにつき1つのブランチを作る
- mainブランチに直接作業しない
- 機能が動いたらPRを作ってmainに統合する
- App.tsxに処理が集まりすぎたらコンポーネント分割する
- 最初から複雑な設計にしすぎない
- MVP完成後にリファクタリングする

---

## 16.注意点

このアプリは、1個目のポートフォリオとしてReactの基礎力を示すためのものである。

そのため、MVP段階では以下を避ける。

- 最初から機能を増やしすぎる
- UIデザインに時間をかけすぎる
- Reduxなどの状態管理ライブラリを入れる
- Next.jsを使う
- バックエンドを作る
- DB設計に踏み込みすぎる

まずは、Reactで小さなアプリを完成させることを最優先にする。
